Why FlashQuant

Production-grade KV cache compression built for real-world LLM serving.

⚡

Fused CUDA Kernels

6 native CUDA kernels fuse compression, decompression, and attention into single passes. Zero intermediate memory allocations.

📈

Split-K FlashDecoding

Parallelize decode attention across KV positions with NUM_SPLITS=4 for maximum SM utilization during autoregressive generation.

📊

O(1) Cache Append

Pre-allocated ring buffers replace O(N²) torch.cat. Cache grows in constant time regardless of sequence length.

🔌

vLLM Backend Plugin

Drop-in attention backend for vLLM v0.18-0.22. No monkeypatching, clean registration with version guards.

🔢

Data-Free Quantization

Random rotation makes coordinates Gaussian, enabling closed-form Lloyd-Max codebooks. No calibration data needed.

🔧

CPU Testable

Pure PyTorch reference implementations for every kernel. Full test suite runs on CPU with no GPU required.

Quick Start

Get up and running in under a minute.

# Install (CPU-only, no compilation needed)
git clone https://github.com/ayinedjimi/flashquant.git
cd flashquant && pip install -e .

# With CUDA kernels
pip install -e ".[dev]"
cmake -B build -DFLASHQUANT_CUDA=ON
cmake --build build -j$(nproc)
import torch
from flashquant import TurboQuantMSE

quantizer = TurboQuantMSE(dim=128, bits=4, seed=42)
x = torch.randn(32, 16, 128) # [batch, heads, dim]

indices, norms = quantizer.quantize(x)
x_hat = quantizer.dequantize(indices, norms)

cos_sim = torch.nn.functional.cosine_similarity(
    x.flatten(0, -2), x_hat.flatten(0, -2), dim=-1
).mean()
print(f"Cosine similarity: {cos_sim:.4f}") # >= 0.95
